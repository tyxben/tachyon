# Tachyon - Product Requirements Document

> 高性能撮合交易引擎 | High-Performance Matching Engine
>
> License: MIT | Language: Rust

---

## 1. 产品愿景

Tachyon 是一个参考 Hyperliquid 架构设计的开源高速撮合交易引擎。目标是构建一个能够达到 **微秒级延迟**、**百万级 TPS** 的纯内存订单撮合系统，适用于：

- 中心化交易所 (CEX) 核心撮合引擎
- 去中心化交易所 (DEX) 链上订单簿
- 高频交易 (HFT) 基础设施
- 金融科技公司的交易系统

### 1.1 项目定位

| 维度 | 说明 |
|------|------|
| **不是什么** | 不是完整的交易所（无前端、无钱包、无链上共识） |
| **是什么** | 纯粹的撮合引擎内核 + 高性能 API 接口 |
| **核心价值** | 极致性能、确定性执行、可嵌入式设计 |

### 1.2 参考基准 (Hyperliquid)

| 指标 | Hyperliquid | Tachyon 目标 |
|------|-------------|---------------|
| 吞吐量 | 200,000 orders/sec | ≥ 5,000,000 orders/sec |
| 中位延迟 | ~100ms (含共识) | < 100ns (纯撮合) |
| P99 延迟 | < 500ms (含共识) | < 10μs (纯撮合) |
| 订单簿类型 | 全链上 CLOB | 内存 CLOB |
| 撮合规则 | 价格-时间优先 | 价格-时间优先 |

> 注：Hyperliquid 的延迟包含共识层开销。Tachyon 作为纯撮合引擎，延迟目标针对撮合逻辑本身。

## 2. 核心功能需求

### 2.1 订单簿 (Order Book)

**P0 - 必须实现:**

- [ ] 中央限价订单簿 (CLOB) - 每个交易对一个独立订单簿
- [ ] 价格-时间优先撮合 (Price-Time Priority)
- [ ] 支持多交易对并发运行
- [ ] 离散价格级别 (tick size) 和最小数量 (lot size)

**支持的订单类型:**

| 订单类型 | 优先级 | 说明 |
|----------|--------|------|
| Limit Order | P0 | 限价单 - 指定价格和数量 |
| Market Order | P0 | 市价单 - 以最优价格立即成交 |
| IOC (Immediate-or-Cancel) | P0 | 立即成交否则取消 |
| FOK (Fill-or-Kill) | P0 | 全部成交否则全部取消 |
| GTC (Good-Til-Cancel) | P0 | 挂单直到取消 |
| Post-Only | P1 | 只做 Maker，不吃单 |
| Stop Limit | P1 | 触发价达到后挂限价单 |
| Stop Market | P1 | 触发价达到后挂市价单 |
| Iceberg Order | P2 | 冰山单 - 只显示部分数量 |
| Trailing Stop | P2 | 追踪止损 |
| TWAP | P3 | 时间加权平均价格 |
| Scaled Order | P3 | 在价格区间内分布多个订单 |

### 2.2 撮合引擎 (Matching Engine)

- [ ] 确定性撮合 - 相同输入必须产生相同输出
- [ ] 原子操作 - 每次撮合要么完全成功要么完全回滚
- [ ] 部分成交支持
- [ ] 自成交防范 (Self-Trade Prevention, STP)
- [ ] 撮合事件流输出 (Trade, OrderAccepted, OrderCancelled, OrderRejected, etc.)

### 2.3 风控模块 (Risk Management)

- [ ] 价格偏离保护 - 防止价格过度偏移
- [ ] 速率限制 - 每账户/每 IP 的下单频率限制
- [ ] 最大订单数量限制
- [ ] 最大持仓限制
- [ ] 断路器 (Circuit Breaker) - 异常波动时暂停交易

### 2.4 API 接口

**P0:**

- [ ] 内嵌库 API (Rust crate) - 零拷贝，直接函数调用
- [ ] gRPC API - 低延迟远程调用
- [ ] 事件流 (Event Stream) - 实时推送撮合结果

**P1:**

- [ ] WebSocket API - 实时行情推送
- [ ] REST API - 管理和查询接口
- [ ] FIX Protocol 4.4 - 金融行业标准协议

### 2.5 可观测性 (Observability)

- [ ] 结构化日志 (tracing)
- [ ] Prometheus 指标 (延迟直方图、吞吐量、订单簿深度)
- [ ] 分布式追踪支持
- [ ] 性能剖析接口

## 3. 非功能需求

### 3.1 性能目标

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 撮合延迟 (中位) | < 100ns | 单次撮合操作 |
| 撮合延迟 (P99) | < 10μs | 极端情况 |
| 撮合延迟 (P99.9) | < 50μs | 毛刺控制 |
| 吞吐量 | ≥ 5M ops/sec | 单交易对 |
| 总吞吐量 | ≥ 20M ops/sec | 多交易对并行 |
| 内存占用 | < 1GB | 单交易对百万级挂单 |
| 启动时间 | < 1s | 冷启动到可接受订单 |

### 3.2 可靠性

- 零内存泄漏 (Rust 所有权保证)
- 零 panic (所有路径必须处理错误)
- 确定性回放 (支持从事件日志恢复状态)
- 快照 + WAL (Write-Ahead Log) 持久化

### 3.3 可扩展性

- 单交易对单线程撮合 (避免锁竞争)
- 多交易对跨核并行 (每个 CPU 核处理独立交易对)
- 无共享架构 (Shared-Nothing)
- 支持水平扩展 (分片)

## 5. 不包含在范围内

以下功能 **不在** Tachyon 核心引擎范围内（可作为独立项目扩展）：

- 用户认证和权限系统
- 资金划转和清结算
- 前端交易界面
- 区块链共识层
- 法币出入金
- KYC/AML 合规模块
